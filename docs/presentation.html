Core Technology

Three Concepts That Power Everything

🧠
Vertex AI (Gemini)
Google's managed AI platform. You rent the brain — Gemini reads and generates human language. You pay per request. We use temperature 0.2: factual, never creative.
Managed · Pay-per-use
📚
RAG
Retrieval Augmented Generation. First retrieve context from YOUR data, then feed it to the AI. Without RAG: generic answers. With RAG: answers about YOUR organization.
Anti-hallucination · Grounded
🔌
MCP Server
Model Context Protocol — open standard by Anthropic. Universal adapter between AI and your data. The AI calls your tools like functions. Model-agnostic: Gemini, Claude, GPT-4 all work.
Open standard · Model-agnostic

"MCP is to AI tools what REST was to web services — a standardization that lets everything connect."

Request Flow

What Happens When You Ask a Question

1
User types: "Which entities have SLA issues?"
Widget sends POST /api/chat → Chatbot API (Spring Boot on Cloud Run)
2
Chatbot API calls MCP tools to fetch context
Calls: get_jargon_dictionary() + get_sla_status() via MCP Server
3
MCP Server routes get_sla_status() → Business Data API
Spring Boot computes compliance % breach risk score, and recommendations using domain logic
4
Chatbot API builds RAG-enhanced prompt
[Jargon Dictionary] + [Live SLA Data] + [User Question] = augmented prompt
5
Vertex AI (Gemini 1.5 Flash) generates grounded answer
Temp 0.2 · System: "Only answer from provided context" · Max 1024 tokens
6
Answer returns to widget — live operational insight in conversational form
"Beta Platform (ENT-002) is BREACHED at 98.5% uptime vs 99.9% threshold. Recommend: immediate escalation."

Security Design

Defense in Depth — Every Layer Protected

👤
Least-Privilege Service Accounts
4 separate service accounts — each with only the permissions it needs. MCP Server gets read-only DB access. Chat API gets only Vertex AI + Secret Manager.
Cloud IAM
🔒
Read-Only Database User
PostgreSQL user `app_readonly` cannot INSERT, UPDATE, or DELETE. Even if the MCP Server were compromised, data cannot be mutated. Defense in depth.
PostgreSQL · Zero mutation risk
🏠
Internal-Only Cloud Run
MCP Server and Business Data API deployed with `--ingress internal`. No public internet access. Only services within the same VPC can reach them.
--no-allow-unauthenticated
🗝️
Secret Manager for All Credentials
DB passwords and API keys in GCP Secret Manager — never in environment variables, never in code, never in Docker images. Fetched at startup only.
Secret Manager · No hardcoded secrets
Architecture Decision

Two MCP Integration Paths — Both Implemented

✅ Option A — Recommended (main branch)
MCP Server (Node.js) calls Spring Boot Business Data API via HTTP REST. Spring Boot stays a standard REST API — no protocol changes.
→ AI calls MCP tool
→ MCP Node.js makes HTTP GET/POST
→ Spring Boot applies business logic
→ Returns JSON to MCP → AI
Lower complexity API reusable
⚡ Option B — Advanced (feature branch)
Spring Boot exposes a /mcp SSE endpoint that speaks MCP protocol directly (JSON-RPC 2.0). Eliminates the Node.js layer for business data.
→ AI connects to /mcp endpoint
→ Spring Boot handles MCP handshake
→ Tools registered via McpSyncServer
→ No Node.js involvement
Java-native Higher complexity
Both options produce identical AI output. The AI never knows which path was used. Switch: git checkout feature/option-b-mcp
